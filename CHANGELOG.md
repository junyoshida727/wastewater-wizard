# CHANGELOG — wastewater-wizard

## [v2.0.0] - 2026-03-10

### 🔄 フロー図（SVG）の大幅改修

- **その他薬品をフロー図に反映** — STEP2で「その他」を選択・台数入力した場合、薬剤名入力済みならその名前で、未入力なら「その他1」「その他2」…としてフロー図上段に表示されるよう対応
- **原水ポンプをフロー図から削除** — 主処理ラインを簡略化
  - 中継槽なし：原水槽 → 凝集沈殿槽
  - 中継槽あり：原水槽 → 中継槽 → 凝集沈殿槽
- **中継槽・監視槽をフロー図に追加** — STEP3の槽オプション選択に連動して動的に表示
  - 中継槽：原水槽と凝集沈殿槽の間に挿入
  - 監視槽：上澄液ライン（凝集沈殿槽→放流）の間に挿入。ろ過装置・放流と縦揃えで配置
- **スラッジ矢印を短縮・修正** — 凝集沈殿槽の真下に垂直に落とす設計に変更。ろ過装置への突き刺さりバグを修正
- **「ろ過しない」選択時のフロー図** — 下段をろ過装置＋放流ではなく汚泥貯槽のみに切り替え。上澄液ラインは維持
- **横スクロールバー復活** — SVGの `width="100%"` を `width="${svgW}"` に修正し、コンテナより広い場合に横スクロールが正しく機能するよう修正
- **PDF印刷時の縮小** — `zoom: 0.62` → `transform: scale(0.55)` に変更し1ページ収まるよう調整

---

### 🗂 ステップ構成の改編

- **STEP3「放流先・規制」→「装置オプション」に変更**
  - 放流先・排水基準の把握の2項目をSTEP2末尾に移動
  - 都道府県選択を削除（納入先都道府県と同一のため）
  - 新たに以下のオプション選択を追加：
    - **槽**（中継槽・汚泥貯槽・監視槽）— 順序：中継槽→汚泥貯槽→監視槽
    - **pH計**（凝集沈殿槽は標準装備として注記。追加設置する槽を選択）
    - **濁度計**（設置槽を複数選択）
    - **脱水機**（必要/不要/未定。「必要」選択時にメーカー・型番の入力欄を展開）

---

### ➕ 新規フィールド追加

- **汚泥量**（STEP2 — 水温の直下）— 単位kg、0.1刻みで入力可能
- **備考・特記事項**（全STEP共通）— STEP1〜5のすべてにテキストエリアを追加。ラベルは「備考・特記事項」に統一
- **レベルセンサー設置箇所を更新** — 旧：原水槽・中間槽・処理水受け槽・ろ過受け槽・その他 → 新：原水槽・中継槽・汚泥貯槽・監視槽・ろ過受け槽・その他

---

### 📱 スマートフォン最適化

- **薬剤詳細テーブル** — スマホ（≤600px）ではヘッダー行を非表示にし、各行をカード形式の縦積みに変換。inputのタップ領域を44px以上に確保
- **脱水機メーカー/型番の2カラム** — スマホでは縦積みに変換
- **フロー図SVG** — `-webkit-overflow-scrolling: touch` を追加し、iOSでも慣性スクロールが機能するよう対応

---

### 🐛 バグ修正

- フロー図のスラッジ水平矢印の終点がろ過装置ノードを突き抜ける問題を修正（`filterX + nodeW/2 - 12` → `filterX - 12`）

## [v1.0.0] - 2026-03-08

### Added
- 5ステップウィザード型ヒアリングシート（基本情報 / 水質情報 / 放流先・規制 / 既存設備 / 現場・施工条件）
- フォーム初期化ボタン（確認モーダル付き）
- localStorage 自動保存 / 下書き再開バナー
- SVGベースのプロセスフロー図（凝集沈殿・フッ素除去・ろ過など条件付き表示）
- PDF出力機能（html2pdf.js）
- モバイル最適化（iOS Safari 対応 / ステップナビ sticky 表示 / タップ領域 44px 以上）
- GitHub Actions による `main` ブランチ → GitHub Pages 自動デプロイ
- `npm run dev` による live-server ローカル開発環境

### Fixed
- iOS Safari のフォーム入力時の自動ズーム（font-size 14px → 16px）
- モバイルでのナビボタンがホームインジケーターに隠れる問題（safe-area-inset 対応）
